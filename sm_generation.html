02 — Pocket Encoder improved over both papers
Multi-resolution SE(3)-EGNN Pocket Encoder
Handles .pdb (60,803 pairs) + .mol2 (3,956 pairs) · outputs h_P ∈ ℝ^{N_P×128} used in cross-attention at every model layer
Atom features
Element (15-hot) · amino acid type (20+UNK) · backbone/sidechain flag · partial charge · H-bond donor/acceptor
Residue features
Secondary structure · SASA (solvent access) · distance to pocket centroid · hydrophobicity · B-factor
Graph build
k-NN k=16 on pocket atoms · RBF distance edge features · dynamic graph per layer · x_P never updated
Output
h_P ∈ ℝ^{N_P×128} · per-atom embeddings · global mean-pool for size prediction · cross-attend into every EGNN layer
⚠ For .mol2 (scPDB): parse with RDKit · map mol2 atom types to same 15-dim one-hot as PDB · define pocket boundary consistently as residues within 8Å of any ligand atom
03 — Flow Matching Backbone replaces 1000-step DDPM
Rectified Flow Matching — Why it beats DDPM for RL
DDPM curved Markov paths make log p(m|pocket) hard to compute → breaks policy gradient · Flow Matching straight paths → tractable ODE likelihood → direct ∇θ log p for DDPO
Forward (training)
z_t = (1−t)·z_data + t·z_noise, t ∈ [0,1] Target velocity: u_t = z_noise − z_data Network: v_θ(z_t, t, pocket) ≈ u_t Loss: E_t[ ||v_θ(z_t,t,P) − u_t||² ] Affinity weighting: w_i = softmax(|aff_i|/T)
Reverse (sampling)
Start: z_0 ~ N(0,I) Euler step: z_{t+Δt} = z_t + v_θ(z_t,t,P)·Δt 50 steps sufficient (20× faster than DDPM) log p via instantaneous change-of-variables: d/dt log p(z_t) = −Tr(∂v_θ/∂z_t)
Coord flow
Continuous · x ∈ ℝ^{N×3} · zero CoM of protein · SE(3)-equivariant updates
Atom type flow
Categorical · {C,N,O,S,F,Cl,Br,P,I,B} · uniform noise mixing · one-hot at t=1
Affinity bias
Higher affinity pairs weighted more in training · flow manifold skewed toward strong binders from pretraining
04 — Unified SE(3)-EGNN with Cross-Attention core model θ
9-Layer SE(3)-Equivariant GNN · hidden_dim=128 · 16 attention heads
Key improvement over both papers: cross-attention from pocket at every layer — not just appended nodes (DiffSBDD) or k-NN edges (TargetDiff)
Per-layer update (ligand atoms)
// 1. Cross-attend to pocket α_i = softmax(Q(h_i)·K(h_P)ᵀ / √128) c_i = V(h_P)·α_i [pocket context] h_i = h_i + LayerNorm(c_i) // 2. EGNN message passing m_ij = φ_e(h_i, h_j, d²_ij, e_ij) ẽ_ij = φ_att(m_ij) [attention gate] h_i' = φ_h(h_i, Σ_j ẽ_ij·m_ij) // 3. Equivariant coord update (SE3 not E3) Δx_i = Σ_j [(x_i−x_j)/d_ij]·φ_d(h_i,h_j,d²,e) + [(x_i−x̄)×(x_j−x̄)] / (||...||+1)·φ_×(...) x_i' = x_i + Δx_i [ligand only, not pocket]
Affinity value head (critic)
// Pooled ligand representation h_pool = MeanPool(h_L over ligand atoms) // Predict affinity (pK proxy) pK_pred = MLP(h_pool) [trained on 64K pairs] // Used as cheap RL reward proxy r_proxy = sigmoid(pK_pred / 16) // Also: entropy of denoised atom types // as secondary affinity signal (TargetDiff insight) r_entropy = −H(softmax(v̂_0))
✓ Value head trained jointly on ALL 64,759 pairs during pretraining · used at every RL step · Vina called only every 10 rounds as ground-truth check
SE(3) equivariance
p(Rx+t|Rx_P+t) = p(x|x_P) · zero CoM on protein · cross-product term breaks E(3)→SE(3)
Stereochemistry
SE(3) not E(3) → reflection-sensitive · distinguishes R/S enantiomers · critical for drug safety
Size prediction
N_L ~ p(N_L|N_P) from global pocket embedding · pocket-size-conditioned prior over #atoms
Featurization
Ligand: element + aromaticity (one-hot) · Pocket: element + residue + backbone · joint embedding MLP

06 — RL Fine-Tuning: DDPO Policy Gradient core novelty vs both papers
DDPO — Denoising Diffusion Policy Optimization through the full flow chain
Both DiffSBDD and TargetDiff never backprop through affinity · DDPO does · this is the primary architectural advantage
DDPO objective
// Treat flow chain as T-step MDP // State s_t = (z_t, pocket) // Action a_t = v_θ(z_t, t, P)·Δt [denoising step] // Reward R given at final step t=1 J(θ) = E_{m~p_θ}[ R(m, pocket) ] - β · KL(p_θ || p_{θ_0}) [stay near pretrain] // Policy gradient: ∇_θ J = E[ Σ_t ∇_θ log p_θ(z_{t+1}|z_t, P) · R ] // KL penalty prevents mode collapse: // β annealed 0.01 → 0.001 over RL training
Training loop
for each RL round: 1. Sample B=32 pockets from RL subset 2. Generate 100 molecules per pocket (50-step flow, no_grad for speed) 3. Score all with R(m, pocket) - proxy r_proxy: every round (fast) - Vina r_vina: every 10 rounds (slow) 4. Select top-10 per pocket as seeds 5. Re-run with grad tracking 6. Compute DDPO loss + KL penalty 7. Update θ with Adam lr=1e-5 8. Curriculum: increase pocket difficulty every 500 rounds
KL regularisation
L = L_RL − β·KL(θ||θ₀) · prevents hallucination · keeps chemistry in drug-like space · β annealed
Curriculum
Start easy pockets (mean affinity −7) → hard (−11) · pocket difficulty = mean affinity of known ligands
Batch strategy
Mix CrossDocked + PDBbind + HiQBind · weight PDBbind 2× later in training (cleaner affinities)
Inpainting RL
Also RL-finetune inpainting: fix scaffold, optimize functional groups toward reward · same θ, different mask
↺ Gradient flows: reward oracle → DDPO loss → ∇θ log p → all 9 EGNN layers + cross-attention + flow velocity head · This loop is absent in DiffSBDD and TargetDiff
07 — Full Training Recipe
Two-phase training on 64,759 pairs
Phase A — Pretraining (200K steps)
Dataset: all 64,759 pairs (clip affinities [−16,0]) Split: protein-level 80/10/10 Loss: L_flow + λ·L_affinity_head L_flow = E[||v_θ − u_t||²] (flow matching) L_affinity = MSE(pK_pred, |vina|/16) λ = 0.1 (balance generative vs critic) Optimizer: AdamW lr=3e-4, β=(0.9,0.999) Batch: 4 complexes (memory: ~16GB) Affinity weighting: w_i = softmax(|aff_i|/2) Hardware: 1× A100 80GB, ~36h
Phase B — RL Fine-tuning (50K steps)
Dataset: top-10% affinity pockets (~5,600 pairs) + full CrossDocked for diversity Loss: L_DDPO = −E[log p(m|P)·R(m,P)] + β·KL(θ||θ_0) Optimizer: Adam lr=1e-5 (10× smaller) Generate: 100 mols/pocket, keep top-10 Vina oracle: every 10 rounds Proxy oracle: every round β schedule: 0.01 → 0.001 (cosine anneal) Hardware: 1× A100 80GB, ~12h additional
